本篇作为 NAS（Network Attached Storage，网络附加存储）的入门说明，帮助你快速理解：

> 目标读者：在家里或小团队希望统一管理文件、备份数据、搭建多媒体服务的运维／技术同学。

## 1. NAS 是什么？

NAS 可以简单理解为「一台专门用来存储和共享文件的迷你服务器」，通过网络对外提供访问能力：

- 通过 SMB/NFS/AFP 等协议挂载到电脑或服务器
- 通过 Web / App 提供多媒体、相册等高级功能
- 常见实现形态：群晖、威联通、自建软路由 + 硬盘阵列等

与直接插在电脑上的移动硬盘相比，NAS 的优势在于：

- **7x24 小时在线**：不依赖某一台个人电脑是否开机
- **多设备共享**：PC、Mac、手机、电视都可以通过网络访问
- **集中备份**：统一管理备份策略和历史版本

## 2. 常见使用场景

### 2.1 家庭照片与视频归档

- 手机自动备份到 NAS，解决云盘容量不足的问题
- 按时间线或相册归档，配合相册应用做 AI 识别
- 重要家庭视频可以做双份备份（NAS + 外置硬盘）

### 2.2 多终端文件同步

- 在 NAS 上创建「团队文档」或「项目资料」共享目录
- 通过桌面客户端将目录同步到多台电脑
- 替代部分「U 盘传文件」「微信传压缩包」的低效操作

### 2.3 媒体中心（影音/音乐）

- 使用 Jellyfin / Emby / Plex 等服务，将 NAS 变成家庭影音中心
- 电视、平板、手机通过客户端播放 NAS 上的电影与剧集
- 可配合下载器（如 qBittorrent、Transmission）构建自动下载 + 自动整理

## 3. 部署与运维注意事项

### 3.1 硬件选型

- 根据容量需求与预算选择合适盘位数（2 盘位、4 盘位、更多）
- 优先选择 NAS 专用或企业级硬盘，提高连续运行的可靠性
- 关注噪音、功耗与散热，家庭环境建议放置在通风位置

### 3.2 网络与权限

- 内网建议使用千兆以上网络，保证多终端同时访问的体验
- 通过用户/组权限划分「家庭共享」「个人空间」「公开资料」
- 如需公网访问，优先走反向代理 / VPN，避免直接暴露管理端口

### 3.3 备份与容灾

- **RAID 不是备份**：RAID 解决的是磁盘损坏问题，而不是误删或勒索
- 建议配置至少一份「异地备份」（如云存储、移动硬盘定期同步）
- 定期演练恢复流程：确保在数据丢失时，知道如何从备份恢复
