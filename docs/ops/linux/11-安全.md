# 11 安全

## 用户安全

基础策略：

- 禁止共享账号。
- 普通工作使用普通用户，必要时 sudo 提权。
- 长期不用账号及时锁定或删除。

常用命令：

```bash
sudo passwd -l testuser      # 锁定用户
sudo userdel -r testuser     # 删除用户及家目录
chage -l app                 # 查看密码策略
```

## SSH 安全

`/etc/ssh/sshd_config` 建议：

```text
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
MaxAuthTries 3
```

修改后重载：

```bash
sudo systemctl reload sshd
```

最佳实践：

- 使用密钥登录，禁用弱密码。
- 限制来源 IP（配合防火墙）。
- 关键主机开启 MFA（跳板机层）。

## 防火墙策略

原则：最小开放。

```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp
sudo ufw status verbose
```

变更建议：

- 先开管理口（SSH），再收紧策略。
- 远程操作时准备回滚任务（如 `at` 定时恢复规则）。

## 文件权限安全

重点目录：

- SSH 私钥：`~/.ssh/id_rsa` 必须 `600`
- SSH 目录：`~/.ssh` 建议 `700`
- 配置文件避免世界可写（`o+w`）

检查高风险文件：

```bash
find / -xdev -type f -perm -0002 2>/dev/null
find / -xdev -type f -perm -4000 2>/dev/null   # SUID
```

## Fail2ban（进阶）

Fail2ban 通过日志识别恶意尝试并自动封禁 IP。

安装与启动（Ubuntu）：

```bash
sudo apt install -y fail2ban
sudo systemctl enable --now fail2ban
```

查看状态：

```bash
sudo fail2ban-client status
sudo fail2ban-client status sshd
```

建议：

- 与 SSH 密钥策略结合使用。
- 白名单办公出口 IP，避免误封。
