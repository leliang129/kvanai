# 04 文本处理

文本处理是 Linux 自动化的核心能力，很多问题都可以通过“过滤 + 提取 + 统计”快速定位。

## `grep`

常见用法：

```bash
grep "ERROR" app.log
grep -n "timeout" app.log
grep -E "ERROR|WARN" app.log
grep -r "listen 80" /etc/nginx
```

高频参数：

- `-n` 显示行号
- `-E` 使用扩展正则
- `-r` 递归目录
- `-v` 反向匹配

## `awk`

`awk` 擅长“按列处理”。

```bash
# 打印第 1 和第 5 列
awk '{print $1, $5}' access.log

# 统计每个状态码数量
awk '{count[$9]++} END {for (code in count) print code, count[code]}' access.log
```

指定分隔符：

```bash
awk -F ',' '{print $1, $3}' users.csv
```

## `sed`

`sed` 擅长“按规则替换或编辑文本流”。

```bash
sed 's/old/new/g' app.conf
sed -n '1,20p' app.log
sed -i 's/DEBUG/INFO/g' app.conf
```

注意：`sed -i` 会直接改文件，先备份再操作。

## `sort/uniq`

```bash
sort names.txt
sort -n numbers.txt
sort access.log | uniq
sort access.log | uniq -c | sort -nr
```

典型用途：

- 统计出现频次最高的 IP、URL、错误码。

## 管道与重定向

管道 `|`：把前一个命令输出作为后一个命令输入。

```bash
cat access.log | grep " 500 " | awk '{print $1}' | sort | uniq -c | sort -nr | head
```

重定向：

- `>` 覆盖写入
- `>>` 追加写入
- `2>` 错误输出
- `2>&1` 合并 stdout/stderr

```bash
./deploy.sh >deploy.log 2>&1
```

实战建议：

- 先用小样本验证表达式，再跑全量。
- 对生产日志优先只读操作，避免误改原始数据。
