# 01 文件系统

## 路径（绝对/相对）

- 绝对路径：从根目录 `/` 开始，例如 `/etc/nginx/nginx.conf`。
- 相对路径：相对当前工作目录，例如 `../logs/app.log`。

常用路径命令：

```bash
pwd        # 当前目录
cd /var/log
cd ..       # 上一级
cd -        # 上一次所在目录
```

## 文件类型

Linux 中“万物皆文件”，常见类型包括：

- 普通文件（`-`）
- 目录（`d`）
- 符号链接（`l`）
- 字符设备（`c`）
- 块设备（`b`）
- 套接字（`s`）
- 管道（`p`）

查看文件类型：

```bash
ls -l
file /bin/ls
```

## 软链接 vs 硬链接

软链接（symbolic link）：

- 本质是“路径引用”，类似快捷方式。
- 可以跨文件系统。
- 原文件删除后软链接失效。

硬链接（hard link）：

- 多个文件名指向同一 inode。
- 不能跨文件系统，通常不能链接目录。
- 删除某个文件名不影响其他硬链接访问。

命令示例：

```bash
ln /data/app.log app.log.hard
ln -s /data/app.log app.log.soft
```

## inode 机制

inode 存储文件元数据（权限、属主、大小、数据块指针），文件名存储在目录项中。

要点：

- 文件名 -> inode -> 数据块。
- 同一个 inode 可能对应多个文件名（硬链接）。
- 磁盘“空间没满但无法创建文件”常见原因是 inode 耗尽。

排查命令：

```bash
df -h   # 查看容量
df -i   # 查看 inode 使用率
```

实战建议：

- 日志分片目录注意小文件爆炸问题。
- 定期清理临时文件和历史归档，避免 inode 耗尽。
