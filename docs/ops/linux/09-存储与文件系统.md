# 09 存储与文件系统

## 挂载（`mount`）

查看当前挂载：

```bash
mount | column -t
lsblk -f
```

临时挂载：

```bash
sudo mount /dev/sdb1 /mnt/data
sudo umount /mnt/data
```

永久挂载通过 `/etc/fstab`：

```fstab
UUID=xxxx-xxxx  /data  ext4  defaults,nofail  0  2
```

修改后建议验证：

```bash
sudo mount -a
```

## 分区

常用工具：`fdisk`、`parted`。

```bash
sudo fdisk -l
sudo parted -l
```

基础流程：

1. 识别新磁盘（`lsblk`）。
2. 创建分区（`fdisk /dev/sdb`）。
3. 格式化文件系统。
4. 挂载并写入 `fstab`。

## 文件系统类型（ext4/xfs）

- ext4：通用、兼容性好，工具成熟。
- xfs：大文件和并发性能较好，企业场景常见。

创建文件系统：

```bash
sudo mkfs.ext4 /dev/sdb1
sudo mkfs.xfs /dev/sdc1
```

查看类型：

```bash
lsblk -f
```

## LVM（进阶）

LVM 适合需要动态扩容的场景。

核心概念：

- PV（Physical Volume）
- VG（Volume Group）
- LV（Logical Volume）

常用命令：

```bash
sudo pvcreate /dev/sdb /dev/sdc
sudo vgcreate vg_data /dev/sdb /dev/sdc
sudo lvcreate -L 100G -n lv_logs vg_data
sudo mkfs.ext4 /dev/vg_data/lv_logs
```

扩容示例：

```bash
sudo lvextend -L +50G /dev/vg_data/lv_logs
sudo resize2fs /dev/vg_data/lv_logs   # ext4
```

实战建议：

- 生产扩容前先快照或备份。
- 关注 IO 等待与文件系统健康（`iostat`、`dmesg`）。
