# 03 权限与用户

## 权限模型（rwx）

Linux 权限分为三组：属主（u）、属组（g）、其他用户（o）。

- `r` 读（4）
- `w` 写（2）
- `x` 执行（1）

示例：`-rwxr-x---` 对应 `750`。

## `chmod` / `chown`

修改权限：

```bash
chmod 644 app.conf
chmod -R 750 /opt/myapp
chmod u+x deploy.sh
```

修改所有者：

```bash
sudo chown root:root /etc/my.cnf
sudo chown -R app:app /srv/myapp
```

## 用户与用户组

常用命令：

```bash
id
getent passwd app
getent group sudo
sudo useradd -m -s /bin/bash app
sudo passwd app
sudo usermod -aG sudo app
```

原则：

- 服务使用专用低权限用户运行。
- 权限按组授予，减少散落到个人账号。

## sudo 机制

`sudo` 用于临时提升权限执行命令，并记录审计日志。

查看 sudo 规则：

```bash
sudo -l
sudo visudo
```

`/etc/sudoers` 推荐通过 `visudo` 修改，避免语法错误导致 sudo 不可用。

最小权限示例（放到 `/etc/sudoers.d/deploy`）：

```text
deploy ALL=(root) NOPASSWD: /usr/bin/systemctl restart myapp
```

安全建议：

- 不要随意给 `NOPASSWD:ALL`。
- 使用 sudo 细化到命令级别。
- 关键主机开启操作审计。
