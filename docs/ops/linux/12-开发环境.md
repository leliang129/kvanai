# 12 开发环境

## Git

常用初始化：

```bash
git config --global user.name "your-name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
```

高频操作：

```bash
git clone <repo>
git checkout -b feature/linux-docs
git add .
git commit -m "docs: add linux learning path"
git push origin feature/linux-docs
```

## 编译工具（`gcc/make`）

安装（Ubuntu）：

```bash
sudo apt update
sudo apt install -y build-essential
```

验证：

```bash
gcc --version
make --version
```

典型流程：

```bash
./configure
make -j$(nproc)
sudo make install
```

## 环境变量

查看与设置：

```bash
env | sort
export APP_ENV=prod
echo "$APP_ENV"
```

持久化建议：

- 用户级：`~/.bashrc`、`~/.zshrc`
- 系统级：`/etc/environment`、`/etc/profile.d/*.sh`

更新 PATH：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

## 包管理（`apt/yum`）

Ubuntu / Debian：

```bash
sudo apt update
sudo apt install -y curl vim git
sudo apt upgrade -y
sudo apt remove <pkg>
```

RHEL / CentOS：

```bash
sudo yum install -y curl vim git
sudo yum update -y
sudo yum remove <pkg>
```

运维建议：

- 固定关键依赖版本，避免不可控升级。
- 在测试环境先验证，再推生产。
- 关注软件源可靠性与签名校验。
