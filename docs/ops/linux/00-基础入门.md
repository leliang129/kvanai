# 00 基础入门

## Linux 简介

Linux 是一个类 Unix 操作系统内核，常见发行版基于 Linux 内核 + GNU 用户态工具构建。运维工作中，Linux 主要承担：

- 服务运行平台（Web、DB、MQ、K8s 节点）
- 自动化脚本执行环境
- 网络与存储管理控制面

核心特点：多用户、多任务、权限模型清晰、工具链丰富。

## 常见发行版（Ubuntu/CentOS/Alpine）

| 发行版 | 特点 | 典型场景 |
| --- | --- | --- |
| Ubuntu | 社区活跃、文档多、`apt` 生态好 | 开发机、通用服务器 |
| CentOS/RHEL 系 | 企业常见、稳定、`yum/dnf` | 传统企业生产环境 |
| Alpine | 体积小、基于 musl libc | 容器镜像、轻量运行时 |

选型建议：

- 通用场景优先 Ubuntu LTS 或 RHEL 兼容发行版。
- 容器镜像可用 Alpine，但要注意与 `glibc` 相关兼容问题。

## 安装与环境准备

最小安装后建议立即完成以下初始化：

```bash
# 1) 更新系统
sudo apt update && sudo apt -y upgrade

# 2) 创建普通用户并加入 sudo（Ubuntu）
sudo adduser devops
sudo usermod -aG sudo devops

# 3) 配置时区和 NTP
sudo timedatectl set-timezone Asia/Shanghai
sudo timedatectl set-ntp true

# 4) 安装基础工具
sudo apt install -y curl wget vim git htop net-tools unzip
```

环境基线建议：

- 禁止直接使用 root 远程登录。
- 保持系统时间同步（日志与证书高度依赖时间准确性）。
- 统一 Shell 与编辑器习惯（如 `bash + vim`）。

## 终端基础

常见终端操作：

- 历史命令：`history`
- 自动补全：`Tab`
- 反向搜索：`Ctrl + r`
- 中断当前命令：`Ctrl + c`
- 后台挂起：`Ctrl + z`

推荐别名（写入 `~/.bashrc`）：

```bash
alias ll='ls -alF'
alias grep='grep --color=auto'
alias ..='cd ..'
```

## 目录结构（`/etc` `/var` `/home`）

- `/etc`：系统与服务配置文件目录，例如 `sshd_config`、`nginx.conf`。
- `/var`：可变数据目录，例如日志、缓存、数据库数据。
- `/home`：普通用户家目录。
- `/usr`：用户态程序与库。
- `/opt`：可选软件安装目录（第三方应用常见）。
- `/tmp`：临时文件，系统可能定期清理。

实战原则：

- 配置看 `/etc`，数据看 `/var`，用户脚本看 `/home/<user>`。
- 改配置前先备份：`sudo cp file{,.bak.$(date +%F-%H%M%S)}`。
