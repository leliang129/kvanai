# 06 进程与系统管理

## `ps/top/htop`

查看进程快照：

```bash
ps -ef
ps aux | grep nginx
ps -eo pid,ppid,cmd,%mem,%cpu --sort=-%cpu | head
```

实时观察：

```bash
top
htop
```

核心关注指标：CPU、内存、负载（load average）、进程状态。

## `kill`

发送信号控制进程：

```bash
kill <pid>        # 默认 SIGTERM(15)
kill -9 <pid>     # SIGKILL，强制终止
kill -HUP <pid>   # 常用于通知进程重载配置
```

建议顺序：先 `TERM`，必要时再 `KILL`。

## 后台任务（`&`/`nohup`）

```bash
./task.sh &
nohup ./task.sh > task.log 2>&1 &
jobs -l
fg %1
bg %1
```

说明：

- `&` 让任务在后台运行，但会受会话影响。
- `nohup` 可在退出终端后继续运行。

## 进程调度

优先级管理：

```bash
nice -n 10 ./batch-job.sh
renice -n 5 -p <pid>
```

定时任务（cron）：

```bash
crontab -e
crontab -l
```

示例：每天 2:30 备份。

```cron
30 2 * * * /usr/local/bin/backup.sh >> /var/log/backup.log 2>&1
```

实战建议：

- 长任务加超时控制与告警。
- 定时任务要记录开始时间、结束时间和结果。
